# Resources: section overview

Working tools that aren't tied to a specific product (TaskFlow or
PayFlow) but get used across any technical documentation project.

## Material in this section

- [Glossary](glossary.en.md) — terms from API documentation, knowledge
  bases, ITIL processes, and GOST standards used throughout this
  portfolio.
- [Checklists](checklists.en.md) — review checklists for API
  documentation, knowledge base articles, incident closure, and release
  notes.
- [Further reading](further-reading.en.md) — standards, style guides, and
  tools I rely on in my work.

## Why a separate "Resources" section

The glossary and checklists aren't finished artifacts like a knowledge
base article or an API reference — they're infrastructure that keeps the
other materials consistent with each other. I keep them in their own
section instead of scattering them across individual examples, for the
same reason shared utilities get pulled into their own module in code:
one place to update, and an obvious place to look.
