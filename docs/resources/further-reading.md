# Дальнейшее чтение

Источники и инструменты, на которые я опираюсь в работе. Список не
претендует на полноту — это то, что реально использовалось при
подготовке этого портфолио и в проектах, которые он представляет.

## Стандарты документирования (РФ/СНГ)

- **ГОСТ 19 (ЕСПД)** — Единая система программной документации. Полезна
  как общий язык для регламентации состава документов (техническое
  задание, руководство пользователя, описание программы), даже там, где
  формально не обязательна.
- **ГОСТ 34** — стандарты на автоматизированные системы. Версии,
  обновлённые в 2020–2021 годах, стоит использовать в приоритете перед
  более старыми редакциями при работе с современными ИТ-проектами.
- **ЕСКД** — Единая система конструкторской документации. Актуальна,
  когда документация касается аппаратной части или изделий, а не только
  программного обеспечения.

## Международные подходы и стиль

- **Google Developer Documentation Style Guide** — практическое
  руководство по стилю для технической документации на английском
  языке, особенно полезно для API-референсов.
- **Microsoft Writing Style Guide** — альтернативный источник стилевых
  рекомендаций с сильным разделом про инклюзивный язык и
  интерфейсные тексты (UX writing).
- **Write the Docs** — сообщество технических писателей; гайды и записи
  докладов с конференций полезны как источник практик, проверенных на
  реальных продуктах.
- **The Good Docs Project** — открытые шаблоны документации (README,
  руководства, справочники по API), которые удобно использовать как
  отправную точку.

## OpenAPI и API-документация

- **Спецификация OpenAPI 3.x** — основной справочник по синтаксису
  спецификации; полезно держать под рукой при ревью чужих
  `openapi.yaml`.
- **Spectral** — линтер для OpenAPI/AsyncAPI-спецификаций; позволяет
  формализовать соглашения об именовании и обязательных полях в виде
  правил, а не устных договорённостей.
- **Redoc / Swagger UI** — инструменты для рендеринга интерактивной
  документации из OpenAPI-спецификации (в этом портфолио используется
  подход, аналогичный `mkdocs-render-swagger-plugin`, см.
  [«TaskFlow API — автогенерируемый референс»](../api-docs/generated-reference/index.md)).

## ITIL и проблем-менеджмент

- **ITIL 4** — источник терминологии и процессной модели для инцидент-
  и проблем-менеджмента (роли, жизненный цикл проблемы, KEDB).
- **Google SRE Book («Site Reliability Engineering»)** — глава про
  postmortem culture — один из самых цитируемых источников по
  практике blameless-разбора инцидентов.

## Инструменты, использованные при сборке этого портфолио

- **MkDocs + MkDocs Material** — генератор статического сайта
  документации и тема оформления.
- **mkdocs-static-i18n** — плагин двуязычной навигации (RU/EN) для
  MkDocs.
- **Pandoc + XeLaTeX (шрифты DejaVu)** — для генерации PDF-версий
  материалов с корректным отображением кириллицы.
- **Stepik** — платформа для публикации обучающих курсов по технической
  документации в формате «урок — шаг».
