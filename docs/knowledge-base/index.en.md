# Knowledge Base: section overview

The examples below are written for the fictional product **TaskFlow** (the
same product used for the auto-generated reference in the
[API Documentation](../api-docs/generated-reference/index.en.md) section) —
but from the perspective of an end user or workspace admin, not a
developer.

## Why a knowledge base, if there's a support team

Every knowledge base article is a support ticket that never had to be
filed. A good knowledge base:

- resolves a question before the user ever contacts support;
- cuts resolution time for support agents — they link an article instead
  of writing an answer from scratch;
- lowers the ramp-up burden for new support hires, since they don't need
  to memorize everything.

## The article structure I use

Every article below follows the same template: **symptom/task → steps →
what to do if it didn't help → related articles**. That's a deliberate
choice — a consistent structure reduces cognitive load both for a reader
who's already stressed about a problem, and for a support agent opening
the article for the third time this shift.
