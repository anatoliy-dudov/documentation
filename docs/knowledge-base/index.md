# База знаний: обзор раздела

Примеры ниже написаны для вымышленного продукта **TaskFlow** (тот же
продукт, для которого в разделе [API-документация](../api-docs/generated-reference/index.md)
показан автогенерируемый референс) — но уже с точки зрения конечного
пользователя и администратора рабочего пространства, а не разработчика.

## Зачем база знаний, если есть служба поддержки

Каждая статья базы знаний — это заявка в поддержку, которая не была
создана. Хорошая база знаний:

- закрывает вопрос до того, как пользователь напишет в поддержку;
- сокращает время решения для агентов поддержки — они дают ссылку вместо
  того, чтобы писать ответ с нуля;
- снижает нагрузку на новых сотрудников поддержки, потому что снимает
  необходимость помнить всё наизусть.

## Структура статьи, которую я использую

Каждая статья ниже построена по одному шаблону: **симптом/задача → шаги →
что делать, если не помогло → связанные статьи**. Это осознанный выбор —
одинаковая структура снижает когнитивную нагрузку на читателя, который
уже нервничает из-за проблемы, и на агента поддержки, который открывает
статью в третий раз за смену.
