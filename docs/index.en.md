# Technical Writer Portfolio

Hi, I'm **Anatoliy Dudov**, a technical writer. This portfolio is built so
you can understand how I structure documents, explain complex technical
concepts, and choose formats across domains — from API references to
incident management — in about 10 minutes.

!!! tip "How to read this portfolio"
    The sections in the left navigation are ordered the way it makes sense
    to read them: **API Documentation → Knowledge Base → Problem Management →
    Resources**. Each example is a complete, self-contained artifact with its
    own context and audience, not a fragment cut from a larger document.

## A bit about me

- I write and structure technical documentation: API references, user
  guides, knowledge bases, and incident/problem management runbooks.
- I develop technical-documentation training courses, including for the
  **Stepik** learning platform.
- I'm well versed in Russian/CIS documentation standards — GOST 19 (ESPD),
  GOST 34, ESKD — and in how they intersect with modern API-documentation
  practices (OpenAPI, docs-as-code, DITA).

More on the [About me](about.en.md) page.
