# API-документация: обзор раздела

В этом разделе — два разных подхода к документированию API. Они закрывают
разные задачи, и в реальной работе я обычно использую оба одновременно.

## 1. Автогенерируемый референс — TaskFlow API

Полный референс эндпоинтов, сгенерированный из **OpenAPI 3.0**-спецификации.
Такой подход хорош, когда:

- API большой и быстро меняется — вручную поддерживать референс дорого;
- нужен единый источник правды (spec) для документации, тестов и клиентских
  SDK;
- важна точность по каждому параметру, коду ответа и схеме объекта.

Моя роль в таком подходе — не писать референс вручную, а:

- проектировать и ревьюить саму OpenAPI-спецификацию (имена, описания,
  примеры, коды ошибок);
- настраивать сборку и линтинг спецификации (например, Spectral);
- писать сопроводительные материалы — концептуальные страницы, гайды,
  которые генератор сделать не может.

## 2. Аннотированная документация — PayFlow API

Документация вымышленного платёжного API в стиле популярных платёжных
провайдеров: быстрый старт, аутентификация, коды ошибок, вебхуки.
Написана вручную, с объяснением решений — то есть каждый раздел
не просто описывает, «что есть», но и «почему это документировано именно
так».

Такой подход нужен, когда:

- API небольшой или средний, и точность формулировок важнее скорости
  обновления;
- аудитория — внешние разработчики, которые интегрируют платежи в первый
  раз и которым нужен понятный путь от «получил ключ» до «первый платёж
  прошёл»;
- часть контента (концепции, edge cases, security-рекомендации) невозможно
  выразить только в OpenAPI-спецификации.

## Как сравнивать эти два примера

| | TaskFlow API (генерируемый) | PayFlow API (аннотированный) |
|---|---|---|
| Источник контента | OpenAPI-спецификация | Markdown, написанный вручную |
| Что хорошо получается | Полнота, консистентность, скорость обновления | Объяснение «зачем», работа с эджкейсами |
| Что не получается | Не объяснит бизнес-логику и контекст | Дороже поддерживать при частых изменениях API |
| Типичный сценарий | Внутренние и партнёрские API с высокой скоростью изменений | Публичный API с высокими ставками на онбординг разработчиков |
